数据库原理及其应用

Created2023-04-23|Updated2023-04-23
|Post View:

第一章

1.1.1
  • 数据
    • 数据库中存储的基本对象
    • 描述事物的符号记录
    • 可以是数字，也可以是文字、图形、图像、声音、语言……经过数字化之后存入计算机
  • 数据库（DataBase/DB）
    • 长期存储在计算机内、有组织的、可共享的大量数据的集合
    • 三个基本特点
      • 永久存储
      • 有组织
      • 可共享
  • 数据库管理系统
    • DataBase Management System/DBMS
    • 数据定义功能
      • DBMS提供Data Definition Language/DDL
      • 用户通过数据定义语言对数据库中的数据对象进行定义
    • 数据组织、存储和管理
      • 组织：怎么放数据才能更高效
      • 存储：怎么把数据存在计算机硬盘里，存到什么位置
      • 管理：怎么调度
    • 数据操纵功能
      • Data Manipulation Language/DML
      • 实现基本操作:查询、插入、删除、修改
    • 事务管理、运行管理
      • 安全性
      • 完整性
      • 多用户对数据的并发使用
      • 故障 系统恢复
    • 数据库建立、维护
      • 初始数据输入
      • 转换
  • 数据库系统：DataBase System
    • 数据库
    • 数据库管理系统
    • 应用系统
    • 数据库管理员
1.1.2 三个发展阶段：人工管理、文件系统、数据库系统
  • 人工管理
    • 特点
      • 数据不保存：纸带
      • 应用程序管理数据：数据和程序绑定
      • 数据不共享
      • 数据不具有独立性
    • 缺点：数据的逻辑结构/物理结构改变后，需要对程序做修改，加重程序员负担
  • 文件系统
    • 数据可以长期保存
    • 由文件系统管理数据
    • 数据共享性差，冗余度大
      • 冗余：重复，丢失易找回；查找工作量增加、容易出错
    • 数据独立性差（只跟一个程序绑定）
  • 数据库系统
    • 数据结构化
    • 共享性高、冗余度低
    • 数据独立性高（不跟应用程序相绑定）
    • 数据由DBMS统一管理和控制
数据模型
  • 数据模型　Data Model
    • 对现实世界的模拟
    • 对现实世界的数据特征的抽象
  • 数据模型要求
    • 真实客观模拟世界
    • 容易被人理解（直观）
    • 便于在计算机上实现
  • 概念模型：用户（什么信息）
    • 实体 Entity：客观存在并可相互区别的事物
    • 属性 Attribute:实体所具有的某一特性
    • 码 Key:唯一标识实体的属性集
      • 例如学号是学生实体的key
    • 域 Domain：属性的取值范围
    • 实体型 Entity Type：
    • 实体集：
    • 联系 relationship
      • 事物内部、事物之间
      • 两个实体型之间的联系
        • 1：1 一对一联系
        • 1：n 一对多联系
        • m：n 多对多联系（一对多是多对多的特例）
          • A对B是一对多，B对A也是一对多
      • 两个以上实体型
        • 也是三种联系
      • 单个实体型内
        • 三种
    • 概念模型表示方法
      • ER图
        • 实体型：用矩形，框内写明实体名
        • 属性：用椭圆形，用无向边与实体型连起来
        • 联系：用菱形，框内写明联系名，用无向边与实体型连起来，并在无向边旁标上联系的类型是几比几
          • 如果联系也有属性，那么也要用无向边联系起来
  • 逻辑模型：DBMS（什么结构）
    • 层次模型
      • 树
    • 网状模型
      • 图、工程流程图
    • 关系模型
      • 本课重点研究
      • 表格就是关系模型
      • 关系 Relation
        • 一个关系就是一张表
      • 元组 Tuple
        • 表中的一行
      • 属性 Attribute
        • 表中的一列
      • 码 Key
        • 能够唯一确定一个元组的属性
      • 域 Domain
        • 属性的取值范围
      • 分量
        • 元组中的一个属性指
      • 关系模式
        • 对关系的描述
      • 关系模型的优点
        • 数学理论基础
        • 概念单一
        • 存取路径对用户透明：安全保密且简化程序员工作（三个模型两级映射）
    • 对象模型
    • 面向对象模型
  • 物理模型：计算机硬件（怎么存储）
组成：数据结构、数据操作、完整性约束三部分
  • 数据结构
    • 数据对象、对象置加的关系
  • 数据操作
    • 查询、插入、删除、更新数据
  • 完整性约束
    • 实体完整性
      • 能够确定唯一实体，不能有歧义
    • 参照完整性
      • 参照其他实体
    • 用户定义的完整性
模式 Schema
  • 三级模式
    • 外模式/应用模式
      • 把数据库里的数据根据用户需求反映给用户
    • 模式/概念模式
      • 具体数据库
      • 二维表
    • 内模式/物理模式
      • 在计算机的实际物理硬盘中如何存储
  • 两级映像
    • 内模式——模式
      • 内模式/模式映像：物理独立性
    • 模式——外模式
      • 外模式/模式映像：逻辑独立性
数据库系统的组成
  • 数据库、数据库管理系统、应用系统、数据库管理员
  • 硬件平台、数据库
    • 硬件资源要求
      • 内存
      • 磁盘
      • 数据处理能力
    • 软件
      • DBMS
      • OS
      • 高级语言、编译系统
      • 应用开发工具
      • 其他
    • 人员
      • 数据库管理员DataBase Administrator/DBA
        • 全面管理和控制数据库系统
          • 决定DB的信息内容和结构
          • 存储结构和存储策略
          • 完全性和完整性约束
          • 监控 使用和运行 日志
          • 改进和重建
      • 系统分析员
        • 需求分析和规范说明
        • 与DBA和用户结合
        • ER图
      • 数据库设计人员
        • 数据的确定 模式
      • 应用程序员
        • 设计和编写应用系统的程序模块，并调试安装
      • 最终用户 End User
        • 通过应用系统的用户接口使用数据库

第二章

发展历程

1970年提出关系数据模型

关系
  • 单一的数据结构
  • 逻辑结构：二维表
  • 集合代数的基础上
  • 基本概念
    • 域 Domain
      • 具有相同数据类型的值的集合
    • 笛卡尔积 Cartesian Product
      • 所有域的所有取值的所有组合
      • 不是笛卡尔积中的所有子集都有意义
    • 元组 Tuple
      • 一行
      • 也叫记录
    • 分量 Component
    • 基数 Cardinal number
      • 这个域里能取几个值
    • 关系 Relation
      • 关系
      • 元组
        • 一般用t表示
      • 单元关系、二元关系
      • 关系的表示：关系就是一张二维表
      • 属性：列
        • 属性的取值范围——域
        • n目关系必有n个属性
      • 码
        • 候选码
          • 凡是能够唯一确定一条记录的属性都叫候选码
          • 属性组
        • 全码 All Key
          • 所有属性才能确定唯一一条记录
        • 主码 Primary Key
          • 从多个候选码中选定其中一个作为主码
        • 主属性
          • 候选码的属性
        • 非主属性
          • 除了主属性的
      • 三类关系
        • 基本关系
          • 实际存储的二维表
          • 在数据库中实际存在的
          • 性质
            • 列 同质
            • 不同属性，属形名不同
            • 列的顺序无所谓，列的次序可以任意交换
            • 行的顺序无所谓，行的次序可以任意交换
            • ==分量必须取原子值==
            • 任意两个元组的候选码不能相同
        • 查询表
          • 查询结果对应的表
        • 视图表
          • 有基本表或其他视图表导出的表
关系模式
  • 关系模式是对关系的描述
    • 关系模式是型，关系是值
      • 元组集合的结构
        • 属性构成
        • 属性来自的域
        • 属性和域之间的映像关系
      • 一个关系通常由赋予它的元组语义确定
      • 现实世界中还存在着完整性约束
        • 和现实世界相一致的约束
  • 定义关系模式
    • R(U,D,DOM,F)
      • R 关系名
      • U 属性名集合
      • D 映射
      • DOM 属性向域的映象集合
      • 属性间的数据依赖关系集合
  • 关系和关系模式
    • 关系模式是静态的
    • 关系是关系模式在某一时刻的状态或内容
    • 在实际工作中往往统称关系，通过上下文区别
关系数据库
  • 关系（二维表）的集合
型与值
  • 型
    • 关系数据库的模式
      • 若干域的定义，以及域上的关系模式
  • 值
    • 关系模式某一时刻的关系的集合
关系模式的存储结构
基本关系操作
  • 查询
    • 选择：选择满足要求的记录
    • 投影：选出列
    • 并
    • 差
    • 笛卡尔积
  • 更新
    • 插入
    • 删除
    • 修改
  • 特点
    • 操作的对象和结果都是集合
  • 语言分类
    • 关系代数语言 ISBL
    • 关系演算语言
      • 元组关系演算 APLHA，QUEL
      • 域关系演算 QBE
    • 具有双重特点的 SQLＳＱＬ
  • 完整性约束
    • 实体完整性
      • 主码不能为空
      • 任意两个记录中主码不能相同
    • 参照完整性
      • 关系间的引用
      • 引用的数据必须完全参照/来源于引用的表里的数据
      • 外码(需要满足三个条件)
        • 属性F不是基本关系R的码
        • F参照另一个基本关系S
        • F是基本关系S的主码
          • R叫参照关系
          • S叫被参照关系
      • 如果是外码,要么取空值,要么等于S中某个元组的主码值
