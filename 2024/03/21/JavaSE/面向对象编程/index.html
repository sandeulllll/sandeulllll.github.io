面向对象编程

面向对象编程

面向过程与面向对象

  • 面向过程和面向对象都是一种思想
  • 面向对象是相对于面向过程而言的
  • 面向过程强调的是功能行为，以函数为最小单位，考虑怎么做
  • 面向对象强调具备了功能的对象，以类/对象为最小单位，考虑谁来做
  • 面向对象的三大特征：封装、继承、多态

类和对象

  • 类是对一类事物的描述，是抽象的、概念上的定义
  • 对象是实际存在的该类事物的每个个体，因而也称为实例。

对象的创建和使用

  • 堆（Heap），所有的对象实例以及数组都要在堆上分配。
    • 也就是说，堆内存用来存放对象实例。
  • 虚拟机栈（Stack）：用于存储局部变量。方法执行完自动释放。
  • 方法区：用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等。

匿名对象

例如：new Person().shout();

  • 经常将匿名对象作为实参传递给一个方法调用
  • 如果对一个对象只需要进行一次方法调用，就可以使用匿名对象

属性（field）

成员变量和局部变量

  • 成员变量：在方法体外、类体内声明的变量
  • 局部变量：在方法体内部声明的变量
  • 比较：
    • 局部变量除形参外，均需显示初始化。
    • 成员变量有默认初始化值。
      • byte，int，short：0
      • long：0L
      • float：0.0F
      • double：0.0
      • char：0
      • boolean：false
      • 引用类型：null
    • 局部变量加载于栈空间。
    • 成员变量加载于堆空间或静态域内。

方法

  • Java里所有的方法必须定义在类里
  • 方法只有被调用才会执行

方法的重载

  • 重载：在同一个类中，允许存在一个以上的同名方法，只要它们的参数个数或者参数类型不同即可。
  • 重载与返回值类型无关，只看参数列表，而且参数列表必须不同。
  • 调用时根据参数列表的不同来区别。

方法参数的值传递机制

  • Java里方法的参数传递方式只有一种：值传递。
  • 值传递：将实际参数值的副本传入方法内，而参数本身不受影响。
  • 两种情况：
    • 形参是基本数据类型：将实参基本数据类型的“数据值”传递给形参
    • 形参是引用数据类型：将实参引用数据类型变量的“地址值”传递给形参

递归方法

  • 递归方法：一个方法调用它自身。

封装和隐藏

封装性

  • 高内聚：类的内部数据操作细节自己完成，不允许外部干涉
  • 低耦合：仅对外部暴露少量的方法用于使用
  • 封装：隐藏对象内部的复杂性，值对外公开简单的接口，便于外界调用，从而提高系统的可拓展性、可维护性。

四种访问权限修饰符

  • private：类内部
  • （缺省）：同一个包
  • protected：不同包的子类
  • public：同一个工程
  • 对于class只有2种修饰符：
    • public类：可以在任意地方被访问
    • default类：只可以被同一个包内部的类访问

构造器（构造方法）

  • 构造器的作用：创建对象；给对象进行初始化。
  • 分类：
    • 隐式无参构造器（系统默认提供）
    • 显示定义一个或多个构造器（无参、有参）
  • Java语言中，每个类至少有一个构造器
  • 默认构造器的修饰符与所属类的修饰符一致
  • 一旦显示定义了构造器，系统不再提供默认构造器
  • 一个类可以创建多个重载的构造器
  • 父类的构造器不可被子类继承

属性赋值的先后顺序

  1. 默认初始化
  2. 显示初始化
  3. 构造器中初始化
  4. 通过对象.属性或对象.方法的方式赋值

JavaBean

  • JavaBean是一种用Java语言写成的可重用组件
  • JavaBean是指符合以下标准的Java类：
    • 类是公共的
    • 有一个无参的公共的构造器
    • 有属性，且有对应的getter和setter方法

this关键字

  • this的作用
    • 在方法内部：表示这个方法所属对象的引用
    • 在构造器内部使用：表示该构造器正在初始化的对象
  • this可以调用类的属性、方法和构造器
  • 可以在构造器汇总使用this(形参列表)来调用本类中重载的其他构造器
  • 构造器中不能通过this(形参列表)来调用自身构造器
  • this(形参列表)必须声明在类的构造器的首行
  • 在类的一个构造器中，最多只能声明一个this(形参列表)

package，import

package

  • 作用：帮助管理大型软件系统，将功能相近的类划分到同一个包中。
  • 包可以包含类和子包
  • 包可以解决类名冲突问题
  • 包可以控制访问权限

MVC 设计模式

model层：业务逻辑判断和数据库存取

  • 数据对象封装 model.bean
  • 数据库操作类model.dao
  • 数据库 model.db

view层：显示数据

  • 相关工具类：view.untils
  • 自定义：view.ui

controller层：将指令和数据传递给业务逻辑

  • 应用界面相关：controller.activity
  • 存放fragment：controller.fragment
  • 显示列表的适配器：controller.adapter
  • 服务相关的：controller.service
  • 抽取的基类：controller.base

JDK中主要的包介绍

  1. java.lang:包含Java语言的核心类，如String，Math，Integer，System和Thread，提供常用功能
  2. java.net：包含执行和网络相关的操作的类和接口
  3. java.io：包含能提供多种输入/输出功能的类
  4. java.util：包含一些使用工具类，如定义系统特性、接口的集合框架类、使用与日期日历相关的函数
  5. java.text：包含一些Java格式化相关的类
  6. java.sql：抱哈了Java进行JDBC数据库编程的相关类和接口
  7. java.awt：包含了构成抽象窗口工具集的多个类，这些类被用来构建和管理应用程序的图形用户界面

import关键字

  • import声明在包的声明和类的声明之间
  • 如果在代码中使用不同包下的同名的类，需要使用类的全类名的方式指明调用的是哪个类

继承

  • Java只支持单继承。
    • 一个子类只能有一个父类，一个父类可以有多个子类
  • 使用extends关键字继承

方法重写

  • 程序执行时，子类的方法将覆盖父类的方法
  • 重写时需要注意：
    • 子类重写的方法必须和父类被重写的方法具有相同的方法名称和参数列表
    • 子类重写的方法的返回值类型不能大于父类被重写的方法的返回值类型
    • 子类重写的方法的方法使用的访问权限不能小于父类被重写的方法的访问权限
    • 子类不能重写父类中声明为private权限的方法
    • 子类方法抛出的异常不能大于父类被重写方法抛出的异常

super关键字

  • 功能：
    • 访问父类中定义的属性
    • 调用父类中定义的成员方法
    • 在子类构造器中调用父类构造器
  • 注意：
    • 子父类出现同名成员时，可以用super表明调用的是父类的成员
    • super的追溯不仅限于直接父类
  • super和this
    • this代表对本类对象的引用
    • super代表父类的内存空间的标识

调用父类的构造器

  • 子类中所有的构造器默认都会访问父类中无参数的构造器
  • 当父类中没有无参数的构造器时，子类的构造器必须通过this访问本类的构造器或通过super访问父类的构造器。且必须放在构造器的首行。
  • 如果子类中既没有显示调用父类或本类的构造器，父类中又没有无参构造器，编译器会报错。

子类对象实例化过程

  1. 产生对象并对成员变量默认初始化
  2. 对构造方法中的形参赋值
  3. super语句调用或隐式调用父类的构造器（一直追溯到Object类为止）
  4. 对当前类中的成员变量显式初始化
  5. 执行当前构造器方法中的代码

多态

  • Java中的多态：父类的引用指向子类的对象
    • 可以直接应用在抽象类和接口上
  • Java引用变量有两个类型（编译时看左边，运行时看右边）：
    • 编译时类型：由声明该变量时使用的类型决定
    • 运行时类型：由实际赋给该变量的对象决定
  • 编译时类型和运行时类型不一致，就体现出了对象的多态性
    • 父类的引用不具备子类特有的方法
    • 实际运行的是子类重写父类的方法
  • 子类的对象可以代替父类使用
    • 一个变量只能由一种确定的数据类型
    • 但一个引用类型变量可以指向多种不同类型的对象
  • 向上转型：
    • 如果一个引用变量声明为父类的类型，但实际引用的是子类对象，那么该变量不能再访问子类中添加的属性和方法

虚拟方法

  • 多态情况下，将父类中与子类同名同参数的方法称为虚拟方法

instanceof操作符

  • x instanceof A:
    • 检验x是否是类A的对象，返回值为boolean类型
  • 如果x所属的类和类A不是子类和父类的关系，编译报错
  • 如果x属于类A的子类B,x instanceof A的结果也为true

对象类型转换（Casting）

  • 基本数据类型：自动类型转换和强制类型转换
  • 对Java对象的强制类型转换：造型
    • 从子类到父类的类型转换可以自动进行
    • 从父类到子类的类型转换必须通过造型实现
    • 无继承关系的引用类型间的转换是非法的
    • 在造型前可以使用instanceof操作符测试一个对象的类型

子类继承父类

  • 若子类重写了父类的方法，子类里定义的方法彻底覆盖父类里的同名同参数方法
  • 但实例变量不是。若子类里定义了与父类完全相同的实例变量，这个实例变量也不会覆盖父类中定义的实例变量

Object类

  • Object类是所有Java类的根父类
  • 如果在类的声明中没有用extends指明其父类，则默认父类是java.lang.Object类
  • Object类主要结构：
    • 构造器public Object（）
    • 对象比较：public boolean equals(Object obj)
    • 取得哈希码public int hashCode()
    • 对象打印时调用：public String toString()

==和equals的区别

  • ==既可以比较基本类型也可以比较引用类型，对于基本类型比较值，对于引用类型比较内存地址
  • equals是java.lang.Object类里的方法，如果该方法没有被重写过也是==
    • String等常用类的equals方法是被重写过的。
  • 通常，重写equals方法，会比较类中的相应属性是否都相等

toString（）方法

  • 进行String和其他类型数据的连接操作时，自动调用toString（）方法
  • 基本数据类型转换为String类型时，调用了对应包装类的toString（）方法

包装类(Wrapper)

  • 包装类：针对8种基本数据类型定义相应的引用类型
    • byte———–Byte
    • short———Short
    • int————-Integer
    • long———-Long
    • float———Float
    • double———-Double
    • boolean——–Boolean
    • char————-Character

static 关键字

  • 实例变量和类变量
    • 实例变量：属于类的每一个对象，不能被同一个类的不同对象共享
    • 类变量：在内存空间里只有一份，被该类的所有对象共享。
  • 类属性和类方法
    • 类属性：不因对象的不同而改变
    • 类方法：与调用者无关，不需要创建对象就可以调用类方法
  • 可以被static修饰的：
    • 属性
    • 方法
    • 代码块
    • 内部类
  • 被static修饰的成员的特点：
    • 随着类的加载而加载
    • 优先于对象存在
    • 被修饰的成员为所有对象共享
    • 访问权限允许时，可以不创建对象而是直接被类调用
  • 方法区：类的加载信息、静态变量、常量
  • 可以用类名.方法名（）调用被static修饰的方法
  • 在static方法内部只能访问类的静态成员。
  • 因为不需要实例就可以访问，所以static方法内部不能有this，也不能有super；
  • static方法不能被重写

单例设计模式：对该类只能存在一个对象实例

  • 私有化构造器
  • 内部提供一个当前类的实例，该实例也静态化
  • 提供公共的静态方法，返回当前类的对象
  • 单例模式的好处：节省性能开销
  • 应用场景：
    • 网站计数器，便于同步
    • 应用程序的日志应用：便于追加内容
    • 数据库连接池
    • 读取配置文件的类
    • Windows的回收站
    • ……

main方法的语法

  • Java虚拟机需要调用main方法，所以是public
  • 执行main方法时不必创建对象，所以是static
  • 接收参数是String类型的数组

类的第四种成员：代码块

  • 代码块的作用：对Java类或对象进行初始化
  • 代码块如果有修饰符，只能是static
  • 分类：
    • 静态代码块
      • 静态代码块的执行要先于非静态代码块
      • 随着类的加载而加载，而且只加载一次
    • 非静态代码块
      • 不仅可以调用非静态结构，还可以调用静态结构
      • 每次创建对象的时候都执行一次，而且优先于构造器执行
  • static代码块通常用于初始化static属性

final关键字

  • 修饰类：不能被继承
  • 修饰方法：不能被子类重写
  • 修饰变量：即常量。
    • 变量名称大写。
    • 只能被赋值一次。

抽象类与抽象方法

  • 用abstract修饰
  • 抽象方法：只有方法声明，没有实现。
  • 含有抽象方法的类必须被声明为抽象类。
  • 抽象类不能被实例化，抽象类是用来继承的。
    • 抽象类的子类必须重写父类的抽象方法，并提供方法体；
    • 若没有子类没有重写所有的父类抽象方法，则该子类仍为抽象类

模板方法设计模式

  • 抽象类作为多个子类的通用模板
  • 当功能内部一部分是确定的，一部分实现是不确定的，可以把不确定的部分暴露出去，让子类去实现。

接口

  • Java不允许多继承，但接口可以得到多继承的效果。
  • 接口：标准，规范，契约。
    • 具备相同的行为特征。
  • 接口是抽象方法和常量值定义的集合。
  • 用Interface定义
  • 所有成员变量默认是public static final
  • 所有抽象方法默认是public abstract
  • 接口没有构造器
  • 接口采用多继承
  • 先写extends，再写implements
  • 一个类可以实现多个接口，接口也可以继承其他接口
  • 实现接口的类必须提供接口中所有方法的具体实现内容，否则仍是抽象类
  • 接口的用途：被类实现（面向接口编程）
  • 从本质上讲，接口是一种的抽象类。
  • jdk及以前：接口只包含常量和抽象方法，没有变量和实现方法。
  • 开发中经常：一个类不是去继承一个已经实现好的类，而是要么继承抽象类，要么实现接口。

jdk8中关于接口的改进

  • 可以为接口添加静态方法和默认方法（合法但违反了接口作为抽象定义的理念）
  • 接口冲突：
    • 若一个接口中定义了一个默认方法，另外一个接口中也定义了一个同名同参数的方法（不管是不是默认方法），再实现类同时实现这两个接口时会出现接口冲突。
    • 解决方法：实现类必须覆盖接口中同名同参数的方法来解决冲突。
  • 类优先原则：一个接口中定义了一个默认方法，父类也定义了一个同名同参数的非抽象方法，不会出现冲突。类优先原则：金额口中具有相同名称和参数的默认方法会被忽略。

内部类

  • 内部类：一个类定义于另一个类的内部
  • Inner class一般用在定义它的类或语句块内，在外部引用时必须给出完整的名称
  • 不能与外层类同名
  • 分类：
    • 成员内部类
      • static成员内部类
      • 非static成员内部类
    • 局部内部类
    • 匿名内部类

成员内部类：

  • 作为类的成员：
    • 可以声明为private或protected
    • 可以调用外部类的结构
    • 声明为static时不能调用外层类的非static成员变量
  • 作为类：
    • 可以定义属性、方法、构造器等结构
    • 可以声明为抽象类
    • 可以声明为final
    • 编译后生成OuterClass$InnerClass.class字节码文件
  • 注意：内部类可以直接访问外层类（包括私有数据），但外层类访问内部类时，需要内部类.成员或内部类对象。成员来访问
  • 想要在外部类的静态成员部分使用内部类时，可以将内部类声明为静态

局部内部类

  • 只能在声明它的代码块或方法中使用
  • 先声明后使用
  • 可以使用外部类的成员（包括私有）
  • 可以使用外部方法的局部变量，但必须是final
  • 不能使用public，protected，缺省，private修饰
  • 不能使用static修饰

匿名内部类

  • 不能定义任何静态成员、方法和类
  • 只能创建匿名内部类的一个实例
  • 一定跟在new后面，隐含实现一个接口或类
  • 特点：
    • 必须继承父类或实现接口
    • 只能有一个对象
    • 只能多态形式引用
  • 格式：
    • new 父类构造器（实参列表）| 实现接口
