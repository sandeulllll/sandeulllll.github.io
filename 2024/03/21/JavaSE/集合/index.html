集合

集合框架概述

  • 两种体系：
    • Collection：单列数据
      • List: 有序，可重复
      • Set: 无序，不可重复
    • Map：
      • 双列数据，保存“key-value”键值对的集合

Collection 接口方法

  • 是List，Set和Queue的父接口
  • 方法（JDK并不提供具体实现，而是提供更具体的子接口）：
    • add 添加
    • size 获取有效元素的个数
    • clear 清空集合
    • isEmpty 判断是否为空
    • contains 是否包含某个元素
    • remove 删除
    • equals 逻辑等
    • toArray 转成对象数组
    • hashCode 获取集合对象的哈希值
    • iterator:用于集合遍历

iterator迭代器接口

  1. 默认游标指向第一个元素之前
  2. Iterator iterator = coll.iterator();
  3. java 5.0 提供了foreach遍历，遍历集合的底层还是用迭代器实现

Collection子接口一：List

通常使用List来替代数组：同样有序可重复，且每个元素有对应的顺序索引

List接口的实现类

ArrayList：引用的可变长的数组

LinkedList：双向链表实现

使用于频繁插入删除

Vector：和ArrayList的区别在于Vector是线程安全的，因此也比ArrayList慢

List接口方法（在Collection接口基础上增加了根据索引操作集合的方法）

Collection子接口二：Set

Set接口也是Collection的子接口，但没有提供额外的方法

  • 如果添加相等的元素，会添加失败
  • Set比较是否相等不是“==”而是equals

Set接口实现类一：HashSet

大多数使用Set集合时都使用HashSet
按哈希算法来存储集合中的元素，因此有很好的存取、查找、删除性能

HashSet特点

  1. 元素无序
  2. 线程不安全
  3. 集合元素可以是null

HashSet集合判断两个元素是否相等：hashCode（）相等且equals（）返回值也相等

放在Set容器中的对象，必须重写equals（）方法和hashCode()方法——“相等的对象必须具有相同的散列码”

Set实现类二：LinkedHashSet：看起来是以插入顺序保存

底层使用红黑树存储数据。

看起来以插入顺序保存是因为使用双向链表维护元素的次序。

Set实现类三：TreeSet：集合元素处于排序状态
