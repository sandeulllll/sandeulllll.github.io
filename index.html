局部变量的作用域只在函数内部，在函数返回后，局部变量的内存已经释放了。
如果函数返回的是局部变量的值，不涉及地址，程序不会出错。
如果返回的是局部变量的地址(指针)的话，程序运行后会出错。因为函数只是把指针复制后返回了，但是指针指向的内容已经被释放了，这样指针指向的内容就是不可预料的内容，调用就会出错。准确的来说，函数不能通过返回指向栈内存的指针(注意这里指的是栈，返回指向堆内存的指针是可以的)。

  • 返回局部自动变量
    • 存储在栈中
    • 函数返回时自动复制一份变量的copy 给调用者
  • 局部静态变量
    • 存储在静态存储区
    • 从初始化一直有效到程序结束
    • 仅分配一次内存
    • 函数返回后变量不会销毁
  • 返回静态的局部变量的地址
    • 没有问题
    • 静态变量的生命周期是整个程序运行时间
  • 返回局部变量的指针
    • 会出错
    • 栈里的变量在超过七作用域或函数返回后就销毁了，不存在了
  • 返回指向常量的字符串指针
    • 存储在常量存储区
    • （不是静态存储区）
    • 函数返回一个指向常量的指针
    • 但返回一个局部字符串的指针不可行

  • 如果返回的是一个局部变量的地址，该局部变量一定要声明为static类型

  • 基本概念

    • 数据 ： 能被计算机识别并能输入计算机中被处理的符号集合
      （data ）换句话说，数据就是计算机化的信息。
    • 数据元素 ：组成数据的有一定意义的基本单位，
      （data element）在计算机中通常作为整体考虑和处理，不再分割。
    • 数据对象：具有相同性质的数据元素的集合，是数据的一个子集。
      （data object）**数据对象可以是有限的，也可以是无限的。
    • 数据结构 ：是指相互之间存在的一种或多种特定关系的数据元素集合 ，
      （data structure) 是带有结构的数据元素结合，它是指数据的组织形式
    • 数据类型 ：是用来刻画一组性质相同的数据及其上的操作的总称。
      （data type）定义了两个集合：
      　数据类型的取值范围和 该类型中可允许 的一组运算 。
      　* 数据类型分类：原子类型 和 结构类型

  • 抽象数据类型

    • 抽象数据类型 ：对具有某种逻辑关系的数据类型进行描述，并在该类型上（abstract data type)进行的一组操作
      它定义了数据对象、数据对象各元素之间 的关系

  • 描述的是一组逻辑上的特性 ，与在计算机内部表示无关。

    • 例如：整数数据类型是一个抽象数据类型，不同的处理器可能实现方法不同，但其逻辑特性相同，即加、减、乘、除等运算是一致的。

  • *通常是用户定义且用以表示应用问题的数据模型，
    通常由基本的数据类型组成，并包括一系列相关服务操作。

  • 是大型软件构造的模块化方法。使用 线性表、栈、队列、串、树、图 等抽象数据类型可以安全、快速、方便地设计功能的大型软件。

  • 抽象数据类型 ：对象的数据类型。它定义了数据对象、数据对象各数据元素之间的关系及对数据元素的操作。

    • 通常指用户定义的解决应用问题的数据模型，
      • 包括数据的定义和操作
    • 体现了程序设计中的问题分解 、抽象 和 信息隐藏 特性。
      • 把实际生活中的问题分解为多个规模小且容易处理的问题，
      • 建立起一个计算机能处理的数据模型
      • 把每个功能模块的实现细节作为一个独立的单元
      • 从而把具体实现过程隐藏起来

  • 抽象数据类型的描述

    • 用一个三元组来表示
      • ADT（D，S，P）
      • D：数据对象集合
      • S：D上的关系集合
      • P: D 的基本操作集合 *　抽象数据类型描述
        数据对象和数据关系的定义用伪代码描述，
    • 基本操作的定义：

//线性表的抽象数据类型描述
ADT List
{
数据对象: D = {$a_i$ | $a_i$$\in$ Elemset, i = 1,2,…,n, n $\geq$ 0}
数据关系：R = {<$a_{i-1}$ ,$a_i$>| $a_{i-1}$, $a_i$ $\in$ D , i = 2, 3,…,n}
基本操作：
InitList(&L)
初始条件：表L不存在
操作结果：构造一个空的线性表
ClearList(&L)
初始条件：表L已经存在
操作结果： 表L被置为空
ListLength(L)
初始条件： 表L已经存在
操作结果： 返回线性表L的元素个数
…
}ADT List

  • 数据的逻辑结构和存储结构

    • 逻辑结构
      • 集合结构
      • 线性结构
      • 树形结构
      • 图结构
    • 存储结构（物理结构）
      • 顺序存储结构
      • 链式存储结构

  • 算法的性能评价

    • 算法定义：描述解决问题的方法
    • 算法特性： 有穷性、确定性、可行性、输入、输出
    • 好的算法（4个层次） ：
      • 正确性
        • 没有语法错误
        • 输入几组数据后能得到满足要求的结果
        • 对于特殊的输入数据能够得到满足要求的结果（衡量标准）
        • 对于一切合法的输入都能得到满足要求的结果
      • 可读性
      • 健壮性
        • 输入数据不合法时，算法也能做出相关处理，而不是产生异常或莫名其妙的结果
      • 高效率和低存储量
  • 算法时间复杂度
    • 事后统计
    • 事前分析估算
      • 算法采用的策略、方法
      • 编译产生的代码质量
      • 问题的规模
      • 书写的程序语言
        • 语言级别越高，执行效率越低
      • 机器执行指令的速度
    • 问题规模
      • 问题大小的表示，一般用整数n表示
    • 算法的时间分析度量标准
      • 算法中所有语句的执行时间总和
        • 不是实际算法的具体运行时间
      • 算法构成
        • 控制结构
          • 顺序、分支、循环
        • 基本语句
          • 赋值语句、声明语句、输入输出语句
      • 算法运行时间：控制结构和基本语句执行时间的总和
      • 一条语句的执行时间
        • 该语句的重复执行次数和执行一次语句所需时间的乘积
      • 语句频度 ： 一条语句的重复执行次数
        • 问题规模为n的语句，
        • 其语句频度表示为f(n)
      • 问题规模
        • 矩阵运算，n为矩阵的阶数
        • 多项式运算，n为多项式的项数
        • 图的有关运算，n为图中顶点的个数
      • for (i = 0; i < n; i++) n
        for (j = 0; j < n; j++) $n^2$
        {
        a[i][j] = 0; $n^2$
        for (k = 0; k < n; k++) $n^3$
        a[i][j] = a[i][k] * a[k][j]; $n^3$
        }
        则总执行次数为f(n) = n + $n^2$ + $n^2$ + $n^3$ + $n^3$ = 2$n^3$ + 2$n^2$ + n
        与 $n^3$ 同阶
  • 算法时间复杂度(算法的时间量度)
    • 对于复杂算法
    • 将算法中的基本操作语句重复执行的频度作为度量标准
  • 基本操作语句：算法中重复执行次数与算法的执行时间成正比的语句
    • 对算法执行时间贡献最大的语句
  • x = x + 1
    • 没有循环或循环次数与问题规模n 无关,
    • 记作 O(1), 称为 常量阶
  • O(n), 线性阶

*　O($n^2$), 平方阶

  • 还有对数阶O($log_2$n), 指数阶O($2^n$)
  • 常用时间复杂度从小到大依次是
    • O(1) < O($log_2$n) < O(n) < O($n^2$) < O($n^3$) < O($2^n$) < O(n!)

  • 空间复杂度分析

什么是数据结构

  • 数据对象在计算机中的组织方式

    • 逻辑结构
      • 线性结构 -> 一对一
      • 树 -> 一对多
      • 图 -> 多对多
    • 物理存储结构

  • 数据对象必定与一系列加在其上的操作相关联

  • 实现这些操作的方法就叫做算法

  • 例1——> 数据的组织方式

    • 如何在书架上摆放图书
      • 新书怎么插入
      • 怎么查找某本指定的书

  • 例2——> 空间的利用效率

    • 打印1到N的所有正整数
      • 循环实现
      • 递归实现
        • 易于自己和他人理解的程序
        • 对空间的占用

  • 例3——>算法的巧妙程度

    • 求给定多项式在定点x处的值

      • 直接翻译实现（慢）
      • 巧妙地运用结合律

    • clock（）

      • 捕捉从程序开始到clock（）被调用时所耗费的时间

      • 单位是clock tick （时钟打点）

      • 可以计算一个函数到底跑了多少秒钟,模板：

      抽象数据类型

      • 数据类型

        • 数据对象集
        • 数据集合相关联的操作集

      • 抽象：

        • 描述数据类型的方法不依赖于具体实现
          • 跟存放数据的机器无关
          • 跟数据存储的物理结构无关
          • 跟实现操作的算法和编程语言均无关

      • 只描述数据对象集和相关操作集是什么

      • 不关心它是怎么做到的（不关心具体实现）

      • 例子 ：关于“矩阵”的抽象数据类型的定义

        • 抽象之处：

          • 对矩阵元素的具体数据类型不关心——> 提高了适用性
          • 不关心矩阵在程序里的存法——>不关心底层细节而专注于顶层的框架逻辑

代数系统一般性质

教材参考：《离散数学》金聪 郭京蕾

  • 代数系统
    • 不同教材上叫法不完全一样
      • 抽象代数、近世代数、代数结构
    • 普通的代数里
      • ==计算的对象==包括但不限于
        • 众多类别的数、多项式、矩阵、命题、集合、图
      • ==计算的方法==也就是运算 包括但不限于
        • 四则运算、逻辑运算、集合运算、矩阵运算
    • 普通代数已经是对客观世界的一种抽象
      • 数学本身就有==高度抽象== 的特点
    • 对众多的运算对象、众多的运算方法
      • 还可以进行==进一步抽象==
      • 可以说是 ==抽象之上的再抽象==
    • 讨论这些运算对象和运算方法
      • 根据它们的相关性质
      • 可以发现很多相似之处
      • 例如德摩根律（逻辑运算和集合运算）
        • 除去符号上的差别，即==运算符的区别==以外
        • 它们的性质其实是一样的
      • 再比如定义幂的运算
        • 数字的幂–变量的幂–关系的幂–方阵的幂
        • 都是幂运算
        • 除了==运算对象不一样==
        • 它是具有相同的性质的
        • 共同点是一般情况下都不满足交换律
          • 幂运算的性质是通过结合律定义出来的
    • 进一步抽象出来一些结论——称为抽象代数
      • 考虑普通的、一般的、广泛的运算对象
      • 定义最一般、最广泛的运算方法
      • 研究它有什么性质
    • 抽象之上的再抽象
      • 不管运算对象集合的具体特性
      • 也不管对象集合上运算的具体意义
      • 只要讨论数学结构的一般特性
        • 并且按照运算所遵循的一般定义和特性
      • 来对这些==数学结构==进行分类研究
    • 这里讨论的数学结构主要指的是代数结构。也就是代数系统
      • 也都是离散的对象
  • 首先==圈定讨论的对象==
    • 把这些讨论的对象放在一起构成一个集合
    • 两大类集合
      • 命题公式构成的集合
      • 谓词公式构成的集合
  • ==对象之间的运算==
    • 命题公式放在一起我们可以根据等价式当中这些规律来==进行运算==，有等价、蕴涵
    • 集合放在一起 交并差补
    • 并且要==分析这些运算所具有的性质==
  • 讨论的内容
    • ==圈定一个集合，划定一个讨论的范围==
    • ==看集合中的元素能进行什么样的运算==
    • 这两点确定下来，就给出了一个代数系统
  • 代数系统是一种数学结构
    • 由集合、关系、运算、公理、定理、定义和算法组成，也称为代数结构
  • ==代数系统的基本概念==
    • 用有序对的形式把它写出来
    • ==< S，* >==
      • 有序对前面第一个分量是 集合S
      • 第二个分量是一种运算符号
    • ==S 和运算 * 需要满足什么样的条件==
      • 代数系统：集合和其上的运算
        • < N, - > 不能构成代数系统
    • 几种典型的代数系统
      • 半群、群、环、域、格、布尔代数
        • 格可以从偏序集的角度来定义，也可以从代数系统的角度来定义
        • 本门课程主要讨论 半群、群、格 这三种
          • 由于时间限制，环和域不在教学计划里
    • 代数系统
      • ==信息安全的理论基础==
        • 尤其是信息安全中的密码学
          • 密码学当前一个研究热点，跟格有关，格密码
    • 代数系统的基本内容（一般概念和一般性质）
      • ==二元运算==
        • 以往的数学概念，数学运算，归纳，分类
        • 结合具体实例，由一般到具体，再由具体到一般
        • 通过一个函数来==定义==二元运算
          • 假设S是一个集合
          • 那么从笛卡尔积S X S 到S 的一个函数
          • 把它称为集合S上的一个二元运算
        • 这个函数写出来实际上是一个二元函数的形式
          • 只不过用了关系当中有序对的形式 f(<x,y>)
            • 因为==定义域==是一个笛卡尔积
              • 笛卡尔积中的元素是有序对
        • 例如 x+y=z 这个加法就是一个二元函数
    • ==如何验证一个运算是否为集合S上的二元运算==
      • S当中任何两个元素都可以进行这种运算 ==可计算==
      • 且运算结果唯一 ==结果唯一==
      • S当中任意两个元素的运算结果都属于S ==封闭性==
