基础知识

机器语言

  • 机器语言：机器指令的集合
  • 机器指令：一台机器可以正确执行的指令
  • 电子计算机的机器指令：一列二进制数字
    • 计算机将其转化为一列高低电平
    • 驱动电子器件
  • CPU：一种微处理器
  • 计算机：由==CPU==和其他受CPU直接或间接控制的==芯片==、==器件==、==设备==组成的计算机系统
  • 微处理器的工作： 电平脉冲控制
    • 机器码

汇编语言产生

  • 汇编语言的主体：汇编指令
  • 汇编指令:记忆友好版本的机器指令😎
  • 计算机只能读懂机器指令
  • 需要将汇编转化成机器指令
    • 编译器
    • ![[Pasted image 20220809115610.png]]

汇编语言的组成

  • 组成
    • 汇编指令:
      • 机器码的助记符
      • 有对应的机器码
    • 伪指令:
      • 没有对应的机器码
      • 由编译器执行
      • 计算机不执行
    • 其他符号
      • 由编译器识别
      • 没有对应的机器码

存储器

  • CPU: 计算机的核心部件
    • 控制整个计算机的运行并进行运算
    • 需要向CPU提供指令和数据
  • 存储器: 存放指令和数据
    • 内存
  • 磁盘: 和内存不同
    • 磁盘上的数据或指令需要被读到内存中才能被CPU使用

指令和数据

  • 指令和数据:
    • 二进制信息
    • 在内存/磁盘上没有区别
  • CPU 工作:
    • 把有的信息看作数据
    • 有的信息看作指令

存储单元

  • 存储器被划分成若干个存储单元

  • 每个存储单元从0开始顺序编号

    • ![[Pasted image 20220809121158.png]]

  • 计算机的最小信息单位: bit

    • 一个二进制位

  • 字节: 8 bit = 1 byte

  • 微型机

    • 存储器的存储单元
      • 一个byte
    • 存储器的容量
      • 最小单位是字节

  • 大容量存储器

    • 1KB = 1024B
    • 1MB = 1024KB
    • 1GB = 1024MB
    • 1TB = 1024GB

CPU对存储器的读写

  • 从内存中读数据
    • 先指定存储单元的地址
      • 就是确定读哪一个存储单元中的数据
  • 一台微机中有很多个器件
  • CPU读写数据时
    • 要指明对哪一个器件进行操作
    • 进行哪种操作
      • 读出数据
      • 写入数据
  • CPU 和外部器件进行的$3$类信息交互
    • 地址信息
    • 控制信息
      • 器件选择
      • 读或写
    • 数据信息
  • CPU 怎样传递信息
    • 信息的本质: 电信号
    • 用导线
  • 总线: 专门连接CPU 和其他芯片的导线
  • 物理上的总线: 一根根导线的集合
  • 逻辑上的总线:
    • 地址总线
    • 控制总线
    • 数据总线
  • 如何命令计算机进行数据读写
    • 向它输入机器码（电平信息）
    • CPU接收机器码后将完成对应的工作

地址总线

  • CPU 是通过地址总线来指定存储器单元的
  • 一根导线可以传送的稳定状态:
    • 高电平$or$低电平
    • 二进制表示是1或0
    • 也就是说一根导线是一位，0或1，两种状态
  • 假如有10根导线
    • 那么可以传递10位二进制数据
    • 10位二进制数可以表示1024个不同的数据
      • 最小数为0，最大数为1023
  • 宽度
    • 一个CPU有N根地址线
    • 就说这个CPU的地址总线的宽度为N
    • 这样的CPU 最多可以寻找2的N次方个内存单元

数据总线

  • CPU 与内存或其他器件之间的数据传送通过数据总线来进行
  • 数据线宽度决定了CPU与外界的数据传送速度
  • 8根数据总线一次可传送一个字节
    • 如果是两字节需要进行两次数据传送
  • 16根数据总线一次可传送两个字节

控制总线

  • CPU 对外部器件的控制通过控制总线
  • 控制总线是总称
    • 实际上一些不同控制线的集合
  • 有多少根控制总线
    • CPU 就能对外部器件提供多少种控制
  • 控制总线的宽度决定了CPU对外部器件的控制能力
  • 内存读或写功能是由几根控制线综合发出的
    • ==读信号输出==的控制线负责由CPU向外传==读信号==
      • CPU 向该控制线上输出低电平表示将要读取数据
    • 有一根==“写信号输出”==的控制线负责==传送写信号==

主板

  • 主板上有==核心器件==和一些==主要器件==
  • 这些器件通过总线相连
  • 这些器件是哪些器件捏?
    • CPU
    • 存储器
    • 外围芯片组
    • 扩展插槽
      • 一般插有 RAM内存条
      • 和各类接口卡

接口卡

  • CPU对外部设备不能直接控制
  • 外部设备
    • 显示器
    • 音箱
    • 打印机
    • 等等qwq
  • 直接控制外部设备的是
    • 插在扩展插槽上的接口卡
  • CPU 直接控制的是接口卡

内存地址空间1)

  • 假如一个CPU的地址总线宽度是10
  • 那么可以寻址1024个内存单元
  • 这1024个内存单元就构成了这个CPU的内存空间

各类存储器芯片

  • 分类
    • 分类依据:
      • 读写属性
        • 随机存储器:RAM
          • 可读可写
          • ==必须带电存储==
          • 关机后存储的内容消失
        • 只读存储器:ROM
          • 只能读取不能写入
          • 关机后内容不消失
      • 功能和连接
        • 随机存储器
          • 存放供CPU使用的大部分数据
        • 装有BIOS的ROM
          • BIOS是什么
            • 一种软件系统
            • 可以对硬件设备进行基本的输入输出
        • 接口卡上的RAM
          • 对大批量输入,输出数据进行暂时存储
            • 例如显卡
        • ![[Pasted image 20220809151256.png]]

内存地址空间2)

  • CPU 控制存储器时
  • 把它们都看作一个由若干存储单元组成的逻辑存储器—–内存地址空间
  • 所有的物理存储器被看作一个逻辑存储器
    • 每个物理存储器占一段地址空间
  • 内存地址空间受CPU地址总线宽度的限制
  • 基于硬件系统编程时
    • 必须知道内存地址分配情况
  • ![[Pasted image 20220809151946.png]]

寄存器

典型CPU

  • 运算器: 信息处理
  • 寄存器: 信息存储
    • CPU中可以用指令读写的部件
    • 程序员通过改变各种寄存器中的内容来实现对CPU的控制
  • 控制器: 控制各种器件进行工作
  • 内部总线: 连接各种器件,传送数据

通用寄存器

  • 通用寄存器: 用来存放一般性的数据
  • 寄存器的逻辑结构
    • ![[Pasted image 20220809152527.png]]

字在寄存器中的存储

  • 字节
  • 字: 1 word = 2 byte
    • ![[Pasted image 20220809153025.png]]

汇编指令

  • 不区分大小写(针对汇编指令/寄存器名字)
  • mov,add

物理地址

  • 物理上都是线性的

16位结构的CPU

  • 处理,传输,暂时存放
  • 16位的地址
  • 一次最多处理16位数据
  • 寄存器最大宽度:16位
  • 寄存器和运算器之间通路:16位

8086CPU给出物理地址的方法

  • 段地址* 16 + 偏移地址
    • 本质含义
      • 基础地址+偏移地址
  • 乘以16,也可以说左移4位
  • ![[Pasted image 20220810152552.png]]

段的概念

  • 内存没有分段
  • 是将若干地址连续的内存单元看作一个段
    • 段地址乘以16 表示段的的起始地址
    • 偏移地址==定义段中的内存单元==

CS和IP

  • CS: 代码段寄存器
  • IP：指令指针寄存器
