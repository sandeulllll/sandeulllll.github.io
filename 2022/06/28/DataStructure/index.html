什么是数据结构

  • 数据对象在计算机中的组织方式

    • 逻辑结构
      • 线性结构 -> 一对一
      • 树 -> 一对多
      • 图 -> 多对多
    • 物理存储结构

  • 数据对象必定与一系列加在其上的操作相关联

  • 实现这些操作的方法就叫做算法

  • 例1——> 数据的组织方式

    • 如何在书架上摆放图书
      • 新书怎么插入
      • 怎么查找某本指定的书

  • 例2——> 空间的利用效率

    • 打印1到N的所有正整数
      • 循环实现
      • 递归实现
        • 易于自己和他人理解的程序
        • 对空间的占用

  • 例3——>算法的巧妙程度

    • 求给定多项式在定点x处的值

      • 直接翻译实现（慢）
      • 巧妙地运用结合律

    • clock（）

      • 捕捉从程序开始到clock（）被调用时所耗费的时间

      • 单位是clock tick （时钟打点）

      • 可以计算一个函数到底跑了多少秒钟,模板：

      抽象数据类型

      • 数据类型

        • 数据对象集
        • 数据集合相关联的操作集

      • 抽象：

        • 描述数据类型的方法不依赖于具体实现
          • 跟存放数据的机器无关
          • 跟数据存储的物理结构无关
          • 跟实现操作的算法和编程语言均无关

      • 只描述数据对象集和相关操作集是什么

      • 不关心它是怎么做到的（不关心具体实现）

      • 例子 ：关于“矩阵”的抽象数据类型的定义

        • 抽象之处：

          • 对矩阵元素的具体数据类型不关心——> 提高了适用性
          • 不关心矩阵在程序里的存法——>不关心底层细节而专注于顶层的框架逻辑
